## DevOps의 진화

2009년 등장한 DevOps는 개발(Dev)과 운영(Ops)의 경계를 허물고 빠른 소프트웨어 배포를 가능하게 했습니다. 그러나 2020년대 들어 조직이 성장하면서 한계가 드러났습니다.

<Callout type="warning">
  **DevOps의 피로감**: 개발자들이 코드 작성 외에도 인프라 관리, CI/CD 파이프라인 유지, 모니터링 등 너무 많은 일을 해야 하는 "DevOps 피로" 현상이 발생했습니다.
</Callout>

## 내부 개발자 플랫폼 (IDP)

IDP(Internal Developer Platform)는 개발자가 셀프서비스로 인프라를 생성하고 배포할 수 있는 플랫폼입니다.

```yaml
# Backstage (Spotify) 컴포넌트 정의 예시
apiVersion: backstage.io/v1alpha1
kind: Component
metadata:
  name: payment-service
  description: "결제 처리 마이크로서비스"
  annotations:
    github.com/project-slug: company/payment-service
    backstage.io/techdocs-ref: dir:.
spec:
  type: service
  lifecycle: production
  owner: payments-team
  system: checkout-system
  dependsOn:
    - component:user-service
    - resource:payments-db
```

## CI/CD 파이프라인 현대화

최신 CI/CD 트렌드:

1. **GitOps**: Git을 단일 진실 공급원(Single Source of Truth)으로 사용
2. **Progressive Delivery**: 카나리 배포, 피처 플래그로 리스크 최소화
3. **Supply Chain Security**: SBOM(소프트웨어 부품 목록) 생성, 코드 서명
